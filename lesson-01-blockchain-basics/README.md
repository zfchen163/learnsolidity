## 🎯 这一课你会学到

- 为什么说它"不可篡改"
- 用 Go 代码模拟一个简单的区块链

## 🤔 人话解释

### 想象一个班级的账本

你们班有一个公共账本，记录谁借了谁多少钱：

```
小明借给小红 10元  - 2024年1月1日
小红借给小刚 5元   - 2024年1月2日
小刚借给小明 3元   - 2024年1月3日
```

**普通账本的问题**：
- 小明可能偷偷改掉"借给小红10元"这条记录
- 账本可能被撕掉一页
- 只有班长能看，其他人不知道真假

**区块链的解决方案**：
1. **每个人都有一份账本**（去中心化）
2. **新记录要大家都同意**（共识机制）
3. **每一页都和上一页连在一起**（链式结构）
4. **改一页就要改后面所有页**（几乎不可能）

### 区块链 = 区块 + 链

```
[区块1] → [区块2] → [区块3] → [区块4]
  ↓         ↓         ↓         ↓
 交易1     交易2     交易3     交易4
 交易5     交易6     交易7     交易8
```

每个区块包含：
- **数据**：这一页记录了哪些交易
- **时间戳**：什么时候记录的
- **上一个区块的指纹**：确保顺序不能乱
- **自己的指纹**：这一页的唯一标识

## 🔑 关键概念

### 1. 哈希（Hash）= 指纹

就像每个人的指纹都不一样，每个区块的数据通过计算得到一个唯一的"指纹"。

```
数据: "小明转给小红10元"
↓ (哈希计算)
指纹: d7e9f1a4b2c5...
```

**特点**：
- 数据改一个字，指纹完全不同
- 无法从指纹推算出原始数据
- 相同数据永远得到相同指纹

### 2. 链式结构

每个区块都记录了上一个区块的指纹：

```
区块1: 指纹=AAA, 上一个=000
区块2: 指纹=BBB, 上一个=AAA  ← 指向区块1
区块3: 指纹=CCC, 上一个=BBB  ← 指向区块2
```

如果有人改了区块1的数据：
- 区块1的指纹变了（AAA → XXX）
- 区块2记录的"上一个=AAA"对不上了
- 整条链断了！

### 3. 为什么难以篡改？

要改区块1的数据，你需要：
1. 改区块1 → 指纹变了
2. 改区块2的"上一个指纹" → 区块2指纹也变了
3. 改区块3的"上一个指纹" → 区块3指纹也变了
4. 改后面所有区块...

而且这一切要在其他人发现之前完成！

## ❓ 常见问题

**Q: 区块链一定要用在加密货币上吗？**
A: 不是！任何需要"防篡改记录"的场景都能用，比如：
- 学历证书
- 供应链追踪
- 投票系统
- 版权保护

**Q: 每个人都有一份账本，不会很占空间吗？**
A: 会的！比特币的完整账本已经超过 500GB。但你可以只保存部分数据（轻节点）。

**Q: 如果我的电脑算力很强，能不能快速改完所有区块？**
A: 这就是"51%攻击"。如果你的算力超过全网50%，理论上可以。但成本太高，得不偿失。
